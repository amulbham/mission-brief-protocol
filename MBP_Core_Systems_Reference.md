### 2.2 Capsule Topology

**Six Primitives**

| TOPOLOGY | DEFINITION | CANNOT DO | DEFAULT FOR |
|----------|-----------|-----------|-------------|
| ISOLATED | Single capsule, single domain | Cannot have mesh edges to siblings · Cannot fork | Single domain, low novelty, no sub-problems |

```

Sub-problem requires independent closure               → MERKLE_CHILD
Domain boundary crossed mid-capsule                   → FORK · Subroutine Swap

```

## APPENDIX — PROOF OF THOUGHT

**PoT — the mark that proves reasoning occurred correctly.**

```
PoT(κ) = SHA-256(canonical_field_state) | closure_condition_met

PoT ≠ confidence · PoT ≠ assertion · PoT ≠ log entry
PoT = binary proof closure occurred
PoT = enforced residue of correct process running through constraint environment
PoT = proof by structural dependency that reasoning traversed the full capsule arc
```

**Three simultaneous functions from one operation:**
1. **Stamp** — binary proof cognitive closure occurred
2. **Chain link** — seals this capsule to its predecessor (parent hash encoded in hash)
3. **Schema completeness gate** — hash cannot generate without all required fields populated

**The chain:**
```
CAP-000 → PoT[INIT] (genesis · no parent · governed exception)
CAP-001 → PoT[A45F] (parent: INIT · encodes prior state)
CAP-002 → PoT[B72C] (parent: A45F · encodes A45F + INIT)
...
CAP-N   → PoT[XXXX] (encodes complete prior chain to genesis)
```

Tamper with any capsule → fingerprint changes → all downstream fingerprints break → immediately detectable.

**Honest ceiling:** Current runtime uses 4-character session-readable identifiers, not live SHA-256 computation. Architecture is correct. Cryptographic enforcement is the next implementation milestone.
